# 화면 정의서

## 1. 문서 개요

- 시스템명: RAG기반 법률 상담 및 문서 요약 시스템
- 문서 목적:
  - 사용자 화면 단위의 역할과 흐름을 정의한다.
  - 각 화면의 접근 권한, 주요 기능, 입출력 요소를 정리한다.
  - 프론트엔드, 백엔드, PM 간 화면 단위 협업 기준으로 활용한다.

## 2. 화면 목록

| 화면명 | 경로 | 접근 권한 | 설명 |
| --- | --- | --- | --- |
| 랜딩 페이지 | `/` | 전체 | 서비스 소개 및 법률 상담 챗봇 진입 화면 |
| 인증 모달 | 모달 | 비로그인 중심 | 로그인, 회원가입, 계정 확인, 비밀번호 재설정 |
| 워크스페이스 목록 | `/workspace` | 프리미엄 이상 | 내가 속한 워크스페이스 목록 및 생성 |
| 워크스페이스 상세 | `/workspace/:group_id` | 그룹 멤버 | 문서, 멤버, 승인, 휴지통 등 협업 허브 |
| 문서 업로드 | `/workspace/:group_id/upload` | OWNER, ADMIN, EDITOR | PDF 업로드 및 업로드 상태 확인 |
| 문서 상세 | `/workspace/:group_id/documents/:doc_id` | 그룹 멤버 | 원본 문서, 요약, 댓글, 검토 정보 확인 |
| 마이페이지 | `/mypage` | 로그인 사용자 | 프로필, 계정, 구독, 보안, 알림 설정 |
| 관리자 페이지 | `/admin` | ADMIN | 사용자, 서비스, 플랫폼 데이터 현황 관리 |

## 3. 화면 상세

### 3.1 랜딩 페이지

- 화면명: 랜딩 페이지
- 경로: `/`
- 접근 권한: 전체
- 화면 목적:
  - 서비스 첫 진입 화면 제공
  - 법률 상담 챗봇 사용 진입점 제공
  - 비로그인 사용자에게 서비스 소개와 회원가입 유도
- 주요 구성 요소:
  - 헤더
  - 소개 섹션
  - 가이드 섹션
  - 채팅 입력 영역
  - 채팅 세션 영역
  - 로그인 사용자용 세션 히스토리
- 주요 기능:
  - 질문 입력 및 답변 수신
  - 로그인 시 채팅 세션 생성/조회/삭제
  - 비로그인 시 챗봇 체험
  - 로그인/회원가입 모달 진입
- 주요 입력:
  - 사용자 질문 텍스트
  - 첨부 문서 또는 워크스페이스 선택값
- 주요 출력:
  - LLM 답변
  - 참고 판례/법률 출처
  - 이전 대화 목록
- 연계 API:
  - `POST /api/chat/sessions`
  - `GET /api/chat/sessions`
  - `POST /api/chat/sessions/{session_id}/messages`
  - `GET /api/chat/sessions/{session_id}/messages`
  - `DELETE /api/chat/sessions/{session_id}`

### 3.2 인증 모달

- 화면명: 인증 모달
- 경로: 모달
- 접근 권한: 전체
- 화면 목적:
  - 로그인 및 회원가입 기능 제공
  - 계정 확인 및 비밀번호 재설정 기능 제공
- 주요 구성 요소:
  - 로그인 폼
  - 회원가입 폼
  - 이메일 인증 코드 입력 폼
  - 계정 확인 폼
  - 비밀번호 재설정 폼
- 주요 기능:
  - 이메일 인증번호 발송 및 확인
  - 회원가입
  - 로그인
  - 계정 확인
  - 비밀번호 재설정
- 주요 입력:
  - 이메일
  - 비밀번호
  - 닉네임
  - 인증번호
- 주요 출력:
  - 로그인 성공 상태
  - 회원가입 완료 상태
  - 인증 성공/실패 메시지
- 연계 API:
  - `POST /api/email/send-verification-code`
  - `POST /api/email/verify-code`
  - `POST /api/auth/signup`
  - `POST /api/auth/login`
  - `POST /api/auth/confirm-account`
  - `POST /api/auth/reset-password`
  - `GET /api/auth/me`
  - `POST /api/auth/refresh`

### 3.3 워크스페이스 목록

- 화면명: 워크스페이스 목록
- 경로: `/workspace`
- 접근 권한: 프리미엄 이상
- 화면 목적:
  - 사용자가 속한 워크스페이스를 확인
  - 워크스페이스 생성 및 진입
- 주요 구성 요소:
  - 워크스페이스 카드 목록
  - 상태 배지
  - 생성 버튼
  - 생성 모달
- 주요 기능:
  - 내 워크스페이스 조회
  - 워크스페이스 생성
  - 삭제 예정 상태 확인
  - 개별 워크스페이스 진입
- 주요 입력:
  - 워크스페이스 이름
  - 워크스페이스 설명
- 주요 출력:
  - 워크스페이스 목록
  - 내 권한 정보
  - 멤버 수, 문서 수, 상태 정보
- 연계 API:
  - `GET /api/groups`
  - `POST /api/groups`

### 3.4 워크스페이스 상세

- 화면명: 워크스페이스 상세
- 경로: `/workspace/:group_id`
- 접근 권한: 그룹 멤버
- 화면 목적:
  - 워크스페이스 내 협업 허브 역할 수행
  - 문서, 멤버, 승인, 휴지통 기능 제공
- 주요 구성 요소:
  - 문서 탭
  - 승인 탭
  - 휴지통 탭
  - 멤버 관리 영역
  - 초대/권한 변경 UI
- 주요 기능:
  - 워크스페이스 상세 조회
  - 멤버 목록 조회 및 초대
  - 멤버 권한 변경 및 추방
  - 삭제 요청 및 삭제 취소
  - 문서 목록/승인 목록/휴지통 조회
- 주요 입력:
  - 초대 대상 사용자명
  - 권한값
  - 검색어
  - 필터값
- 주요 출력:
  - 워크스페이스 상세 정보
  - 멤버 목록
  - 문서 상태 목록
  - 삭제 예정 상태
- 연계 API:
  - `GET /api/groups/{group_id}`
  - `DELETE /api/groups/{group_id}`
  - `POST /api/groups/{group_id}/cancel-delete`
  - `GET /api/groups/{group_id}/members`
  - `POST /api/groups/{group_id}/members`
  - `PATCH /api/groups/{group_id}/members/{target_id}`
  - `DELETE /api/groups/{group_id}/members/{target_id}`
  - `POST /api/groups/{group_id}/members/{target_id}/transfer`
  - `POST /api/groups/{group_id}/leave`
  - `GET /api/groups/{group_id}/documents`
  - `GET /api/groups/{group_id}/documents/pending`
  - `GET /api/groups/{group_id}/documents/approved`
  - `GET /api/groups/{group_id}/documents/rejected`
  - `GET /api/groups/{group_id}/documents/deleted`

### 3.5 문서 업로드

- 화면명: 문서 업로드
- 경로: `/workspace/:group_id/upload`
- 접근 권한: OWNER, ADMIN, EDITOR
- 화면 목적:
  - PDF 문서를 워크스페이스에 업로드
  - 업로드 및 요약 처리 상태 모니터링
- 주요 구성 요소:
  - 파일 드롭존
  - 업로드 대기 목록
  - 진행 상태 목록
  - 승인 담당자 선택 UI
  - 요약 현황 카드
- 주요 기능:
  - 파일 선택 및 드래그 앤 드롭
  - 업로드 파일 검증
  - 승인 담당자 지정
  - 업로드 실행
  - 처리 상태 polling
- 주요 입력:
  - PDF 파일
  - 승인 담당자
- 주요 출력:
  - 업로드 성공/실패 상태
  - 문서 처리 상태
  - 요약 완료 건수
- 연계 API:
  - `POST /api/groups/{group_id}/documents/upload`
  - `GET /api/groups/{group_id}/documents`
  - `GET /api/groups/{group_id}/members`

## 4. 공통 화면 정책

- 인증이 필요한 화면은 로그인 상태에서만 접근 가능해야 한다.
- 관리자 전용 화면은 ADMIN 권한만 접근 가능해야 한다.
- 워크스페이스 관련 화면은 그룹 멤버십 및 역할 검증이 필요하다.
- 파일 업로드 화면은 PDF 형식과 용량 제한 정책을 따라야 한다.
- 장시간 작업은 로딩 상태와 처리 결과를 사용자에게 명확히 표시해야 한다.
