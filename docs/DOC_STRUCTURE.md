# 문서 구조 규칙

## 1. 기본 원칙

- 제출/운영 기준으로 유지하는 문서만 저장소에 둡니다.
- 작업 메모, 점검 기록, 리팩토링 이력은 최종 산출물에서 제외합니다.
- 구조가 바뀌면 기준 문서를 현재 코드 기준으로 다시 씁니다.

## 2. 위치 규칙

| 위치 | 역할 |
| --- | --- |
| `docs/` | 제출/운영 기준 문서 |
| `backend/docs/` | 백엔드 기술 문서 |
| `frontend/STRUCTURE.md` | 프론트 구조 보조 문서 |

## 3. 현재 기준 문서

### `docs/`

- `01_요구사항정의서.md`
- `02_기능구조도.md`
- `03_화면정의서.md`
- `04_ERD.md`
- `05_프로그램목록.md`
- `10_사용자매뉴얼.md`
- `11_운영자매뉴얼.md`
- `README.md`
- `deploy.md`

### `backend/docs/`

- `ARCHITECTURE.md`

## 4. 갱신 규칙

- 실행 계약이 바뀌면 `README.md`, `deploy.md`, `11_운영자매뉴얼.md`를 먼저 갱신합니다.
- 구조가 바뀌면 `05_프로그램목록.md`, `04_ERD.md`, `backend/docs/ARCHITECTURE.md`를 갱신합니다.
- 제출 시점에는 작업 메모성 문서가 남아 있지 않아야 합니다.
