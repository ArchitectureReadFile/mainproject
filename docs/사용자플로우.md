# 판례 요약 시스템 - 사용자 플로우

이 문서는 현재 `/Users/ijiyun/team` 코드 기준으로 작성한 사용자 플로우 문서입니다.

## 1. 서비스 개요

- 사용자는 PDF 판례 문서를 업로드할 수 있습니다.
- 서버는 업로드된 문서를 백그라운드에서 요약 처리합니다.
- 사용자는 업로드된 판례를 검색하고 상세 내용을 열람할 수 있습니다.
- 관리자 사용자는 관리자 페이지에 접근할 수 있습니다.

## 2. 사용자 유형

- 비로그인 사용자
  메인 페이지 접근 가능
- 일반 사용자 (`GENERAL`)
  로그인 후 업로드, 검색, 상세 조회, 프로필 접근 가능
- 관리자 (`ADMIN`)
  일반 사용자 기능 + 관리자 페이지 접근 가능

## 3. 비로그인 사용자 플로우

```text
메인 페이지 (/)
├── 서비스 소개 확인
├── 업로드/문서 목록 등 보호 페이지 접근 시
│   └── ProtectedRoute에 의해 로그인 필요
└── 인증 모달 열기
```

## 4. 인증 플로우

### 4-1. 로그인

```text
로그인 모달
├── 이메일 입력
├── 비밀번호 입력
├── /api/auth/login 요청
│   ├── 실패: 에러 메시지 표시
│   └── 성공: access_token + refresh_token 쿠키 설정
└── /api/auth/me로 사용자 정보 복원
```

현재 구현 기준:
- 로그인 시도 횟수 초과 시 Redis 기반 rate limit 적용
- 액세스 토큰 만료 시 Axios 인터셉터가 `/api/auth/refresh`를 1회 재시도

### 4-2. 회원가입

```text
회원가입 모달
├── 이름 입력
├── 이메일 입력
├── 인증번호 발송
│   └── /api/email/send-verification-code
├── 인증번호 확인
│   └── /api/email/verify-code
├── 비밀번호 / 비밀번호 확인 입력
└── /api/auth/signup 요청
```

현재 구현 기준:
- 이메일 인증 완료 후에만 회원가입 가능
- 비밀번호는 최소 8자

### 4-3. 가입 계정 확인

```text
계정 확인 모달
├── 이메일 입력
├── 인증번호 발송
├── 인증번호 확인
└── /api/auth/confirm-account 요청
    └── 이메일 / 닉네임 표시
```

### 4-4. 비밀번호 재설정

```text
비밀번호 재설정 모달
├── 이메일 입력
├── 인증번호 발송
├── 인증번호 확인
├── 새 비밀번호 입력
└── /api/auth/reset-password 요청
```

## 5. 로그인 사용자 플로우

### 5-1. 로그인 상태 복원

```text
앱 시작
└── /api/auth/me 요청
    ├── 성공: AuthContext에 user 저장
    └── 실패: 비로그인 상태 유지
```

### 5-2. 업로드 플로우

경로: `/upload`

```text
업로드 페이지
├── PDF 파일 선택 또는 드래그 앤 드롭
│   ├── PDF만 허용
│   └── 최대 5개까지 허용
├── 업로드 세션 생성
│   └── POST /api/documents/upload-session
├── 파일별 업로드 시작
│   └── POST /api/documents/upload
├── 문서 상세 polling
│   └── GET /api/documents/{doc_id}
├── 완료 시
│   └── 처리 현황 카드에서 사건번호/법원/판결일/요약 표시
└── 실패 시
    └── 실패 메시지 표시
```

현재 구현 기준:
- 파일 업로드는 프론트에서 순차 처리
- 업로드 세션 상태는 Redis에 저장
- 페이지 이탈 시 대기 중 항목은 실패 처리
- 재진입 시 업로드 세션 복구 가능

### 5-3. 업로드 이탈 / 복구 플로우

```text
업로드 처리 중
├── beforeunload 감지
├── pagehide 감지
├── /upload 라우트 이탈 감지
└── 업로드 세션 abandon 처리
    └── POST /api/documents/upload-session/abandon
```

현재 구현 기준:
- 처리 중인 1건은 계속 진행될 수 있음
- 나머지 waiting 항목은 failed 처리
- 이후 `/api/documents/upload-session`으로 상태 복구

### 5-4. 문서 목록 / 검색 플로우

경로: `/documents`

```text
문서 목록 페이지
├── 목록 로드
│   └── GET /api/documents
├── 키워드 검색
├── 보기 필터
│   ├── my
│   └── all
├── 카테고리 필터
└── 페이지네이션
```

현재 구현 기준:
- 검색 기준은 `original_filename`, `summary_text`
- 삭제 버튼은 카드에서 소유자 또는 관리자에게만 노출

### 5-5. 문서 상세 플로우

경로: `/documents/:id`

```text
문서 상세 페이지
├── GET /api/documents/{doc_id}
├── 요약 내용 표시
├── PDF 다운로드
│   └── GET /api/summaries/{summary_id}/download
└── 삭제
    └── DELETE /api/documents/{doc_id}
```

현재 구현 기준:
- 삭제 버튼은 업로더 본인 또는 관리자에게만 표시
- 삭제 시 확인 모달 사용

### 5-6. 프로필 플로우

경로: `/profile`

현재 화면 기준 주요 기능:
- 사용자 정보 조회
- 업로드 파일 목록 확인
- 닉네임 변경
- 계정 관련 설정 접근

주의:
- 프로필 페이지에서 사용하는 일부 계약은 별도 추가 검증이 필요할 수 있음

## 6. 관리자 플로우

경로: `/admin`

```text
관리자 페이지
└── ADMIN 사용자만 접근 가능
```

현재 구현 기준:
- 관리자 페이지는 존재함
- 상세 운영 기능(API 연동 범위)은 화면 구현과 실제 백엔드 범위를 별도 검토하는 것이 안전함

## 7. 로그아웃 플로우

```text
로그아웃 시도
├── 업로드 중이면 확인창 노출
├── logout guard clearSession 실행
├── POST /api/auth/logout
└── 메인 페이지로 이동
```

## 8. 현재 구현 기준 주의사항

- 업로드 성공 후 원본 PDF는 `uploads/`에 남아 있습니다.
- 업로드 실패 후 원본 PDF는 삭제를 시도합니다.
- 문서 목록 검색 시 summary가 없는 문서의 keyword 동작은 별도 검토 포인트가 있습니다.
- 업로드 세션은 Redis TTL 기반으로 유지됩니다.
